Hemlock Framework: Docker Cleanup & Best Practices

Overview

The Hemlock core framework image must always include the OpenClaw gateway, Hermes MCP (agent brain/loop, auto-learning), and a shared read-only skills/ directory. Each agent/crew is an independent container with its own persistent volume, mounted at runtime. This ensures the framework is always ready, and agent/crew state is isolated and portable.

Required Changes

  • Root Dockerfiles: Only bake in the gateway, MCP, agent loop, entrypoint, and shared skills. Do not copy agent/crew data into the image.
  • docker/ and docker/hermes-agent/: Remove all test scripts, benchmarks, datagen, optional skills/scripts not used, and docs. Keep only Hermes agent code, MCP, entrypoint, patches, and shared skills.
  • Entrypoint: Must dynamically detect and initialize agent/crew data at runtime, not at build time.
  • Volumes: Each agent/crew gets its own persistent volume, mounted at runtime, and only removed manually by the user.

How to Accomplish It

  1. Update your Dockerfile to not copy agent/crew data. Only copy code, entrypoint, and skills.
  2. Prune docker/ and docker/hermes-agent/ to remove unnecessary files (tests, docs, unused scripts).
  3. Ensure your entrypoint.sh scans for agent/crew directories at runtime and initializes them if needed.
  4. When running containers, mount agent/crew directories as volumes:
    docker run -v /host/agents/agent123:/data/agents/agent123 ...
  5. To remove an agent/crew, manually delete its volume/directory from the host.

Why These Changes Are Correct

  • Ensures the framework is always ready to accept/import any agent or crew without rebuilding the image.
  • Keeps agent/crew state isolated, persistent, and portable.
  • Reduces image size and attack surface by removing unnecessary files.
  • Follows Docker best practices for separation of code and data.

Examples & Templates

Minimal Dockerfile (root)

FROM python:3.11-slim

# Install system dependencies
RUN apt-get update && apt-get install -y --no-install-recommends \
    curl git jq build-essential && rm -rf /var/lib/apt/lists/*

# Copy Hermes agent and OpenClaw gateway code
COPY docker/hermes-agent /app/hermes-agent
COPY entrypoint.sh /app/entrypoint.sh
COPY skills/ /app/skills/
COPY patches/ /app/patches/

WORKDIR /app
RUN pip install --no-cache-dir ./hermes-agent
RUN python3 /app/patches/add_extras.py
RUN chmod +x /app/entrypoint.sh

ENTRYPOINT ["/app/entrypoint.sh"]

Minimal Entrypoint Logic

#!/bin/bash
# Dynamically detect agent/crew ID from mounted volume or environment
if [ -z "$AGENT_ID" ]; then
  # Try to infer from mounted directories (example: first subdir in /data/agents)
  AGENT_ID=$(ls /data/agents | head -n 1)
fi
HERMES_HOME="/data/agents/${AGENT_ID}"
mkdir -p "$HERMES_HOME"
# ... (rest of init logic)
python3 -m hermes_cli.main gateway run &
if [ -f "/app/agent_brain_mcp.py" ]; then
  python3 /app/agent_brain_mcp.py --brain &
fi
wait

Docker Run Example

docker run -d \
  -v /host/agents/:/data/agents/ \
  -v /host/skills:/app/skills:ro \
  -e AGENT_ID= \
  openclaw/framework:latest

Pruning Unnecessary Files

# Remove tests, docs, and unused scripts
rm -rf docker/hermes-agent/tests
rm -rf docker/hermes-agent/benchmarks
rm -rf docker/hermes-agent/datagen*
rm -rf docker/hermes-agent/docs
rm -rf docker/hermes-agent/optional-skills
