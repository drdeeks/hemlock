## Snap Debug — v11 Fix (Route-Based Navigation)
**Root cause:** snap-hono POST handler strips query params. Buttons targeting `/?action=gate_pass` never passed the action to the handler.

**Fix:** Route-based navigation (like the snap template). Each action gets its own registered path:
- `/` — gate page (GET)
- `/gate-pass` — pass button (POST)
- `/gate-yass` — yass button (POST)
- `/swipe-pass?tid=X` — swipe pass (POST)
- `/swipe-yass?tid=X` — swipe yass (POST)

**Also fixed:**
- Replaced unicode arrows (⬅➡) with snap icon props (`icon: "arrow-left"`, `icon: "arrow-right"`)
- Unicode chars were stripped by the snap renderer

**Lesson learned:** Follow the snap template pattern — route-based navigation, not query params. The snap-hono library registers separate routes for each handler.

**Debug protocol used:** debug-tracer — traced the full POST flow layer by layer, found the issue at the "backend receipt" layer (query params not passed).

## Snap Debug — v13 Fix (Image URL + Icons)
**Issue 1:** Image URL error — Neynar pfp URLs from `imagedelivery.net` lack file extensions. Snap renderer rejects them: "Image URL must end with a supported extension (jpg, jpeg, png, gif, webp)".

**Fix:** `safeImageUrl()` function:
- Validates URL against supported extensions
- For `imagedelivery.net` URLs: appends `?format=webp` (forces webp output)
- Falls back to a reliable placeholder image

**Issue 2:** Button icons (`arrow-left`, `arrow-right`) may not render on all Farcaster clients.

**Fix:** Removed icon props from buttons. Clean text labels only.

**Lesson:** Farcaster snap renderer is strict about image URL format. Always validate URLs before returning them in snap UI.

## Snap Debug — v14 Fix (Image URL Extension Validation)
**Issue:** Farcaster client (Warpcast) validates image URLs — must end with .jpg/.jpeg/.png/.gif/.webp. Neynar's imagedelivery.net URLs end in `/original` — no extension. Query param `?format=webp` doesn't help because the client checks the URL path, not the query string.

**Root cause:** Client-side validation, not snap-side. The snap JSON is valid but the Farcaster renderer rejects extensionless image URLs.

**Fix:** `safeImageUrl(url, username)` function:
- If URL path ends with valid extension → use as-is
- Otherwise → generate personalized placeholder: `https://dummyimage.com/200x200/1A1528/C4A7E7.png?text=<initials>`
- Dark purple bg (#1A1528) + lavender text (#C4A7E7) matches theme

**Lesson:** Farcaster snap image URLs MUST end with a file extension in the path. Neynar's CDN URLs are incompatible. Use placeholder service or proxy.

**Snap doc reference:** https://docs.farcaster.xyz/snap/llms.txt — full spec for future reference.
